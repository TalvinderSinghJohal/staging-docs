## Navigating the Home Page

Once logged in, you'll be directed to the **Dashboard**, the control center of your AltrumAI experience.

### The Home Page Includes:
- **Navigation Panel** – Quick access to Dashboard, Administration, Reporting, Observability, Support, and Documentation.
- **Summary Statistics** – Key metrics, charts, and graphs on projects, requests, and policy violations.
- **Project List** – A searchable and filterable table view of all projects with actionable controls.

![Screenshot: AltrumAI Home Page UI]

## Creating and Configuring a New Project

### Step 1: Launch the Project Creation Flow

1. Click the **Create Project** button in the left-hand navigation panel  
   _or_ click the small blue **+ icon** in the bottom-right corner of the screen.
2. You will be redirected to the **Create Project** page.

![Screenshot: Create Project button and icon]

---

### Step 2: Enter Project Details

1. Fill in the required fields:
   - **Project Name** (required)
   - **Project Description** (required)
   - **Project Owner** (required)
   - **LLM Service Providers** (select one or more)
2. Optionally, add:
   - **Project Tags**
3. Click **Create & Continue**.

![Screenshot: Create Project form with example fields filled]

---

### Step 3: Configure the Project

Upon creation, you'll be redirected to the **Project Setup** environment. This contains three tabs:

#### 3.1 Configure Policies

1. Navigate to the **Configure Policies** tab.
2. From the list of available policies, click to select a policy.
3. Use the panel on the right to:
   - Enable or disable the policy
   - Set enforcement mode (e.g., **Log Only** or **Log and Override**)
   - Adjust thresholds or select relevant criteria (topics, entities, regulations)

![Screenshot: Policy configuration interface]

> **Note:**  
> For more detail on individual policies, refer to the [Policy Configuration Guide](/product-guide/policies).

#### 3.2 Configure Response

1. Set response parameters to guide the LLM’s behavior.
2. These configurations determine tone, scope, or fallback responses depending on your use case.

![Screenshot: Configure Response tab]

#### 3.3 Test Policies

1. Use the built-in sandbox environment to:
   - Interact with a test chatbot
   - Validate how policies behave in real time
   - Review flagged prompts and responses
2. Once satisfied, click **Continue to Integration** to proceed.

![Screenshot: Test Policies interface]

---

### Step 4: Configure Integration

1. Select an **Integration Type** from the available options.
2. Once selected, your **API Key** will be displayed.
3. Click **Continue to Summary**.

![Screenshot: Integration step with API key display]

---

### Step 5: Review and Publish

1. Review your project’s setup on the **Summary Page**.
2. Use the edit icons to make changes to any step if needed.
3. Once confirmed, click **Publish Project**.
4. A confirmation message will appear: _“Project successfully published”_.
5. Click **View Project** to open the **Project Dashboard**.

![Screenshot: Project Summary screen with Publish Project button]

---

## Project Dashboard

Once a project is published, the **Project Dashboard** becomes the central view for all configuration, activity, and policy enforcement related to that project.

Key sections include:
- **Project Details**
- **Connection Status**
- **Policy Summary**
- **Integration Details**
- **Usage and Violation Metrics**

![Screenshot: Project Dashboard overview]

---

## Editing Project Details

### To Edit an Existing Project:

1. Navigate to the **Dashboard**.
2. In the **Project Table**, find your project and click the **ellipsis (⋮)** in the **Actions** column.
3. Select **View Project**.
4. On the **Project Dashboard**, locate the **Project Details** quadrant.
5. Click **Edit** to return to **Step 1** of the setup process.
6. Modify the necessary fields and click **Save Changes**.

![Screenshot: Editing project from dashboard with ellipsis menu]

---

## Best Practices

- Use **Test Policies** to validate behavior before going live.
- Start with **Log Only** mode to monitor policy performance without disruption.
- Apply **Log and Override** to sensitive projects for stricter compliance.
- Regularly review **Project Dashboard metrics** for enforcement insights.
- Use descriptive names and tags to improve search and reporting.
